### 标准注意力
```python
# x: [B, T, D]

att = (q @ k.transpose(-2, -1)) * (self.head_dim ** -0.5)
mask = self.local_mask[:, :, :T, :T]
att = att.masked_fill(mask == 0, float("-inf"))
att = F.softmax(att, dim=-1)
att = self.attn_drop(att)
y = att @ v
```
>[!note] 时空复杂度
>**时间复杂度**  $O(T^2D)$
>**空间复杂度** $O(BHT^2)$  H为多头注意力个数
>
### 线性注意力
标准causal attention 对token t:
$$y_t = \sum_{i\leq t}\text{softmax}(q_t\cdot k_i^\top) v_i = \frac{\sum_{i\leq t}e^{(q_t\cdot k_i^\top) v_i}}{\sum_{i\leq t}e^{(q_t\cdot k_i^\top)}}$$
Linear 将$e^{qk^\top}$ 用$\phi (q)^\top \phi(k)$ 近似。

这里定义函数F.elu(x) + 1.0
则：
$$y_t = \frac{\sum_{i\leq t}e^{(q_t\cdot k_i^\top) v_i}}{\sum_{i\leq t}e^{(q_t\cdot k_i^\top)}} = \frac{\sum_{i\leq t}\phi(q_t)^\top \phi(k_i)v_i}{\sum_{i\leq t}\phi(q_t)^\top \phi(k_i)} = \frac{\sum_{i\leq t} \phi(k_i)v_i}{\sum_{i\leq t} \phi(k_i)}$$
提出了qt项，则将复杂度变成了线性复杂度。

```python
@staticmethod
def kernel(x: torch.Tensor) -> torch.Tensor:
	return F.elu(x) + 1.0

q = self.kernel(q)
k = self.kernel(k)
# [B, H, T, D, D]
kv = torch.einsum("bhtd,bhte->bhtde", k, v)
kv_prefix = kv.cumsum(dim=2)
k_prefix = k.cumsum(dim=2)
numerator = torch.einsum("bhtd,bhtde->bhte", q, kv_prefix)
denominator = torch.einsum("bhtd,bhtd->bht", q, k_prefix).unsqueeze(-1)
y = numerator / (denominator + self.eps)
```
>[!note] 时空复杂度
>**时间复杂度** $O(D^2T)$
>**空间复杂度** $O(BHTD^2)$  长序列场景下T>>D
>

### 分析
![[Pasted image 20260430185118.png]]![[Pasted image 20260430185135.png]]
![[Pasted image 20260430185149.png]]
三种模型参数量均为 0.834M，因为它们使用相同的 QKV 投影层、输出投影层、MLP、Embedding 和 LayerNorm，区别仅在 attention forward 过程中的计算公式不同，因此不会改变可学习参数量。
从理论复杂度看，标准 Softmax Attention 需要显式计算 QK^T，其中 Q,K ∈ R^{T×d}，因此单个 attention head 的注意力得分计算复杂度为 O(T^2d)，之后的 softmax 和与 V 相乘也需要 O(T^2) 和 O(T^2d) 的开销。因此总体复杂度约为 O(T^2d)，显存复杂度为 O(T^2)。

然而在本实验中，Softmax Attention 的实测推理速度高于 Linear Attention。这并不否定上述复杂度分析，主要原因是 TinyShakespeare 实验使用的序列长度较短，且标准 softmax attention 的核心操作 QK^T 和 AttnV 是高度优化的 GPU 矩阵乘法；而本文的 Linear Attention 是PyTorch 实现，包含 einsum、cumsum 等未融合操作，并会产生 [B,H,T,d,d] 的中间张量。因此在短序列和未优化实现下，Softmax Attention 由于算子优化和常数开销更小，反而取得了更高的实测速度。该结果说明 Linear Attention 的理论优势主要体现在长序列和高效 kernel 实现条件下。

在困惑度方面，Softmax Attention 的 PPL 为 8.50，Linear Attention 的 PPL 为 9.12，说明基于核函数的线性近似会带来一定建模性能损失。Hybrid Attention 的 PPL 为 8.38，与 Softmax Attention 接近，说明层间混合结构可以在保留部分精确注意力建模能力的同时引入线性注意力。

### 长序列测试
针对上述传统softmax注意力速度大于线性注意力的情况，单独进行了长序列测试，结果如图：
![[Pasted image 20260430185920.png]]
实验结果显示，在短序列 T≤2048 时，Softmax Attention 由于底层矩阵乘法优化充分，实测速度快于或接近 Linear Attention。但随着序列长度增加，Softmax Attention 的时间和显存开销快速增长。当 T=4096 时，Linear Attention 开始反超；当 T=8192 时，Linear Attention 的推理时间为 10.16 ms，而 Softmax Attention 为 20.14 ms，Linear Attention 约快 1.98 倍。同时，Softmax Attention 的峰值显存为 1234.3 MB，而 Linear Attention 仅为 214.3 MB，说明 Linear Attention 在长序列场景下具有明显的速度和显存优势。
