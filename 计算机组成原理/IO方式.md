
# 程序查询方式

数据交换由CPU执行程序实现
### 流程：

1. CPU初始化参数，预置传送参数（起始地址，数据量等）
2. 向io接口发送命令字，启动io设备
3. 读取外设状态寄存器，**直到设备就绪**
4. 执行数据传送
5. 修改地址和计数器参数
6. 转3，直到计数器归零。（数据传送完成）

对于3， 有**独占查询**（忙等待）和**定时查询**两种方式。

# 程序中断方式

基本思想: 进程发起IO操作，启动相应外设，进入阻塞状态，CPU随即转到另一个进程工作。因此，IO和CPU并行工作。当IO准备完成，发起中断，CPU保存当前进程工作环境，跳转到中断处理程序，处理IO请求，处理完了，再回来继续工作。

## 中断工作流程

### 中断请求

可屏蔽中断（INTR）
不可屏蔽中断（NMI信号），优先级最高，不受中断允许标志的影响。
### 中断响应判优先

中断屏蔽技术

在中断响应判优之前，进行中断屏蔽。实现灵活的优先级设置。
### CPU响应中断

条件:
1. 存在有效中断
2. CPU开中断
3. CPU指令运行结束。

### 中断响应过程

1. 关中断。Cpu响应中断之后，首先关闭中断允许标志。
2. 保存断点
3. 引出中断服务程序。
	每个中断源被分配一个唯一的中断类型号，对应一个中断服务程序的**入口地址**，称为**中断向量
	
### 中断处理过程
1. 关中断
2. 保护现场
3. 判断中断源->中断优先级判断
4. 开中断，此时允许更高优先级的中断打断。
5. 执行中断服务程序
6. 关中断
7. 恢复现场
8. 开中断
9. 中断返回指令

# DMA方式

完全由硬件控制的数据传输方法

DMA控制器

DMA数据传送方式：
1. 停止CPU访问主存
2. DMA与CPU交替访问主存（分时法）
3.  周期挪用

传送过程：
1. 预处理
2. 数据传送
3. 后处理

## DMA与中断的区别

DMA的优先级比程序中断高。 

1. DMA不需要保存现场
2. 中断请求在每个指令执行结束后相应，而DMA请求在每个总线周期结束即可相应

# 通道控制方式

#### 通道的类型
1. 字节多路通道，以字节交叉方式传送信息。
	**字节多路通道的计算**
2. 选择通道
3. 数组多路通道

**通道程序执行结束时引起的中断: I/O中断**
A. 访管中断：用户程序请求操作系统服务时产生，如系统调用。
B. I/O中断：I/O设备或通道完成操作、出错等情况产生。
C. 程序性中断：程序执行出错产生，如除零、溢出、非法指令。
D. 外部中断：来自 CPU 外部的事件，如定时器中断、控制台中断等。

问题：
1. 采用硬件而不是软件保存并更新程序计数器的原因：**为了保证中断响应的快速性和可靠性**
2. 中断触发器保存外设中断请求，是因为**CPU无法对发生的中断请求立即进行处理**
3. 硬件状态
4. 目态，管态:
5. 目态 = 用户态
管态 = 核心态 / 系统态 / 特权态



| 缩写      | 全称                   | 中文名   | 保存的内容       | 作用                       |
| ------- | -------------------- | ----- | ----------- | ------------------------ |
| **PC**  | Program Counter      | 程序计数器 | 下一条要执行的指令地址 | 指挥CPU下一步去哪里取指令           |
| **IR**  | Instruction Register | 指令寄存器 | 当前正在执行的指令   | 暂存刚取来的指令，供CPU译码和执行       |
| **PSW** | Program Status Word  | 程序状态字 | CPU当前状态标志   | 记录运算结果状态、中断允许状态、CPU工作模式等 |
| **AR**  | Address Register     | 地址寄存器 | 当前要访问的内存地址  | 向内存提供读/写地址               |
